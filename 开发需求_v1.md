# 《微光苔原 / Sporelight》开发需求 v1

更新时间：2026-05-18

## 1. 版本目标

v1 先完成一版可以打通游戏流程的网页游戏原型，重点验证“命名 → 主菜单 → 关卡选择 → 游戏内滚动与播种 → 结算 → 图鉴解锁”的完整闭环。

本版本不追求三关完整谜题深度，也不追求最终美术精修；优先保证页面跳转、核心交互、存档状态、素材接入和基础动效都可运行。

## 2. 项目定位

- 游戏名称：《微光苔原》
- 英文名：Sporelight
- 主角默认名：露薇 / Luvie
- 类型：治愈系解谜冒险网页游戏
- 核心关键词：安全感、微小史诗、诗意交互
- 情绪基调：没有失败、没有惩罚、没有倒计时；错误操作用柔和提示反馈。

## 3. v1 最小游戏流程

1. 启动页
   - 展示游戏 Logo、背景氛围和进入反馈。
   - 首次进入或无玩家名时进入命名页。
   - 已有玩家名时可进入主菜单。

2. 命名页
   - 默认名称为“露薇”，开发时统一存储为玩家自定义名。
   - 支持输入、随机名字、恢复默认、开始旅程。
   - 名称校验通过后写入本地存档，并跳转主菜单。

3. 主菜单
   - 展示背景、Logo、开始游戏/继续旅程、图鉴、设置入口。
   - 开始游戏进入关卡选择。
   - 继续旅程进入最近已解锁或最近游玩的关卡。
   - 图鉴进入图鉴页。
   - 设置 v1 按补充设计稿和图素开发弹窗。

4. 关卡选择
   - 展示 3 个主题关卡入口：苔藓山谷、雨滴池塘、星光洞穴。
   - v1 仅要求第 1 关可完整游玩。
   - 第 2、3 关展示锁定状态，避免流程断裂。

5. 游戏界面
   - 第 1 关实现一个短流程谜题：滚动到目标区域，在可播种点播种，生成可通行/可达成目标的植物，抵达出口。
   - 支持鼠标点击目的地移动，键盘方向键或 WASD 作为补充控制。
   - 支持播种按钮与键盘 E 触发播种。
   - 完成目标后进入结算页。

6. 结算页
   - 展示通关结果、收集/发现条目、继续/重玩/返回关卡选择。
   - 写入关卡完成状态。
   - 解锁至少 1 个图鉴条目。

7. 图鉴页
   - 展示已解锁和未解锁条目。
   - v1 至少包含：萌芽花台、发光苔藓、藤蔓桥、发光小虫、睡莲、光菇、遗迹拱门、星花。
   - 已解锁条目可查看简短详情，未解锁条目显示问号/锁定。

## 4. 页面需求

### 4.1 启动页

- 参考设计稿：`设计稿_v1/启动页.png`
- 画布：1280×720，16:9 等比适配。
- 页面元素：
  - 背景氛围图。
  - 游戏 Logo。
  - 轻微呼吸光效或孢子浮动。
  - 点击/按任意键进入。
- 跳转规则：
  - 无 `playerName`：进入命名页。
  - 有 `playerName`：进入主菜单。

### 4.2 命名页

- 参考设计稿：`设计稿_v1/输入玩家名.png`
- 素材目录：`assets/images/sporelight_naming_page_assets`
- 功能：
  - 输入框默认值：露薇。
  - 随机名字：从预设名称库中随机填入。
  - 恢复默认：恢复为露薇。
  - 开始旅程：校验名称并保存。
- 校验规则：
  - 不允许空名。
  - 中文建议 1-6 个字。
  - 英文/数字建议 1-12 个字符。
  - v1 可先限制为中英文、数字和常见空格，过滤明显特殊符号。
- 存储：
  - `sporelight.playerName`
  - `sporelight.hasNamed`
- 随机名称库：
  - 露薇、茸茸、小伞、渺渺、芽芽、绯儿、朵朵、团子
  - 糯糯、微光、孢子、苔米、青苔、露珠、花崽、绒球

### 4.3 主菜单

- 参考设计稿：`设计稿_v1/主菜单.png`
- 素材参考：
  - `backgrounds/bg_mainmenu.png`
  - `ui/logo/ui_logo_cn_en.png`
  - `ui/buttons/ui_btn_start.png`
  - `ui/buttons/ui_btn_continue.png`
  - `ui/buttons/ui_btn_gallery.png`
  - `ui/buttons/ui_btn_settings.png`
- 功能：

  - 继续旅程：存在进度时进入最近关卡；无进度时置灰或进入关卡选择。
  - 图鉴：进入图鉴页。
  - 设置：按补充设计稿和图素展示设置弹窗，包含基础设置能力。

### 4.4 关卡选择

- 参考设计稿：`设计稿_v1/关卡选择.png`

  - 1-3 星光洞穴：v1 锁定。

- 基础交互：
  - 点击地图可移动到目标点。
  - WASD/方向键可移动。
  - 到达播种点附近时，播种按钮点亮。
  - 点击播种按钮或按 E 播种。

- v1 谜题建议：
  - 玩家从左下出生。
  - 中间有 2 个播种点。

  - 第 2 个播种点长出藤蔓桥。
  - 玩家通过桥抵达右上出口。
- 操作反馈：
  - 点击位置显示涟漪。
  - 移动时显示短暂孢子拖尾。
  - 不可播种时点击播种给出柔和晃动/提示，不扣分。

## 5. 核心系统需求

### 5.1 场景路由

- 建议使用前端状态机或轻量路由管理页面：
  - `boot`
  - `naming`
  - `mainMenu`
  - `levelSelect`
  - `game`
  - `result`
  - `gallery`
  - `settings`
- 页面切换统一走淡入淡出或 `fx_transition.png`。

### 5.2 存档

v1 使用 `localStorage` 即可，建议统一封装：

```json
{
  "playerName": "露薇",
  "hasNamed": true,
  "lastLevel": "valley",
  "levels": {
    "valley": {
      "unlocked": true,
      "completed": false,
      "bestStars": 0
    },
    "pond": {
      "unlocked": false,
      "completed": false
    },
    "cave": {
      "unlocked": false,
      "completed": false
    }
  },
  "gallery": {
    "moss": true,
    "vine": true
  },
  "settings": {
    "bgm": 0.7,
    "sfx": 0.8
  }
}
```

### 5.3 输入控制

- 鼠标/触控：
  - 点击/触摸目标点移动。
  - 点击按钮触发播种、返回、提示等。
- 键盘：
  - WASD/方向键移动。
  - E 播种。
  - Esc 返回或打开暂停/设置弹窗。
- v1 移动可采用 2D 平面坐标，不强求真实物理引擎。

### 5.4 角色状态

- `idle`：待机呼吸。
- `move`：滚动帧循环。
- `sow`：播种序列。
- `celebrate`：关卡完成或共生反应。

### 5.5 关卡对象状态

- 播种点：
  - `empty`
  - `sowed`
  - `grown`
- 通路对象：
  - `locked`
  - `opening`
  - `open`
- 出口：
  - `inactive`
  - `active`
  - `completed`

## 6. 美术与适配要求

- 主画布基准：1280×720。
- UI 安全区：左右 40px，上下 30px。
- 适配：保持 16:9 等比缩放，优先兼容 1024×576。
- 视觉风格：
  - 低多边形、柔和阴影、几何自然。
  - 莫兰迪色系加温暖微光。
  - 角色红色菌盖作为视觉焦点。
  - 页面元素需要有轻微呼吸、漂浮或光晕动画。
- PNG 素材优先使用现有切图；缺少的文字或布局可先用 CSS/HTML 实现。

## 7. 音频需求

v1 可先预留音频系统，不强制提供最终音频资源。

- 背景音乐：循环播放，支持开关和音量。
- 音效事件：
  - 点击 UI。
  - 角色移动。
  - 播种。
  - 植物生长。
  - 解锁图鉴。
  - 通关结算。

如果暂时没有音频素材，可用静默占位，代码结构保留事件接口。

## 8. 技术建议

当前需求适合使用 HTML5 前端实现：

- 技术方案：Vite + TypeScript + Canvas。
- 页面 UI 可用 DOM/CSS，游戏主场景使用 Canvas。
- 初版游戏逻辑可采用 2D Canvas 坐标与轻量碰撞，不引入复杂物理引擎。
- 素材路径建议统一从 `assets/images` 下配置，不在组件中散落硬编码。

## 9. v1 不包含范围

- 不做完整三关谜题内容。
- 不做复杂物理引擎。
- 不做在线账号、云存档、排行榜。
- 不做完整敏感词库，仅做基础输入校验。
- 不做最终音频制作。
- 不做移动端竖屏单独布局，仅做 16:9 等比适配。

## 10. 验收标准

- 首次打开能进入命名页，输入名称后进入主菜单。
- 刷新页面后能保留玩家名和进度。
- 主菜单、关卡选择、游戏界面、结算页、图鉴页都可正常跳转。
- 第 1 关可以完成：
  - 角色可移动。
  - 至少 2 个播种点可触发。
  - 至少 1 个生长物改变通路或关卡状态。
  - 抵达出口后进入结算页。
- 结算后写入关卡完成状态，并解锁至少 1 个图鉴条目。
- 图鉴页能区分已解锁/未解锁状态。
- 页面在 1280×720 和 1024×576 下无明显文字溢出、按钮错位或核心内容裁切。
- 主要素材来自现有 `assets/images`，并符合全局设计规范。

## 11. 建议开发顺序

1. 搭建项目结构与资源路径配置。
2. 实现基础页面容器、16:9 适配和场景路由。
3. 实现存档模块。
4. 实现启动页、命名页、主菜单。
5. 实现关卡选择页。
6. 实现第 1 关游戏场景与角色移动。
7. 实现播种、生长、通路开启、出口完成。
8. 实现结算页与图鉴解锁。
9. 实现图鉴页。
10. 补充动效、按钮状态、设置弹窗和基础音频接口。
11. 浏览器自测并记录到 `开发进度.md`。
